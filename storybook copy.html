Dark Theme Storybook

Every color, type, and spacing token from sonicwall-cc-tokens v1.2.0, rendered live and mapped to its Tailwind utility class. Values are unchanged from the source theme — this is a documentation and hand-off surface, not a redesign. Each component below carries a property → token → value → Tailwind-class table for the dev team.

Tailwind token map

How the W3C tokens map onto the Tailwind theme namespace — files: tailwind.tokens.js (v3) · tailwind.theme.css (v4)
Source token groupTailwind theme keyExample utility
color.surface.*colors.surface.{l0,l1,l2,l3,hover,sink}bg-surface-l2
color.border.*colors.border.{DEFAULT,subtle}border-border-subtle
color.text.*colors.content.{primary,secondary,muted,…}text-content-secondary
color.accent.*colors.accent.{DEFAULT,hover,subtle,ring,secondary}bg-accent · ring-accent-ring
color.semantic.*colors.severity.{critical,mitigated,…}text-severity-mitigated
typography.fontFamily.*fontFamily.{base,semi,bold,light,display,mono}font-bold
typography.fontSize.*fontSize.{xs…4xl} (with lineHeight)text-base
typography.letterSpacing.*letterSpacing.{wide,wider,widest}tracking-widest
border.radius.*borderRadius.{card,input,pill,badge}rounded-pill
spacing.*spacing.{topbar-h,sidebar-w,footer-h,…}h-topbar-h
Naming note: the source color.text.* group maps to the Tailwind content namespace (so classes read text-content-primary rather than the ambiguous text-text-primary). Every hex / size value is identical to the source — only the key names differ.
